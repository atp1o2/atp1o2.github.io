Classes Class

ft. Google Wallet

5/30/15

Let’s use Google Wallet as our model to explain classes. The application itself will act as our class that holds all of our methods to gather data and methods to output what we want. Keep in mind this isn’t how Google Wallet actually works, but hopefully will serve as a nice metaphor for classes! So let’s say I wanted to add my debit card to Google Wallet. I can call a method, say “Add”, that will ask for the my debit cards credentials (which I won’t add here). After inputting the credentials, Google Wallet will store my debit card, call it DEBIT_1, as an instance of that class. It looks like I entered the wrong address, so I can call a method, “Change Card”, to change my address. The new address will still be saved into the DEBIT1 instance. Now say I add another DEBIT2 card. The beauty of classes is that all of its instances are held under the class, so they can interact with each other. For example, if I wanted to “Transfer”, I could tell DEBIT1 to talk to DEBIT2 and then transfer the money.
If you were the programmer of this class, you would be coding the “Add”, “Change Card”, and “Transfer” methods of this class for other users to use. If you were the user, you can access this class by just calling the methods. Hope this helps as the bare basics for classes!
